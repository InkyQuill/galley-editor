# v0.7.0 — Public Repository Cleanup

## Goal

Prepare Galley Editor's source tree for public visibility by removing local reference source, deleting pre-rebrand image leftovers, replacing throwaway Storybook image fixtures with Galley brand assets, and preserving a concise audit trail for future editor UX work.

## Status

**shipped 2026-04-30**

## Scope

### Remove local reference source

The local `3rdparty/editor/` tree is removed from the repository. Galley is a CodeMirror markdown editor, not a full multi-engine editor package, and future work should be specified directly for Galley's architecture.

### Preserve the audit trail

`docs/specs/editor-reference-audit.md` records:

- what Galley already implements;
- what was intentionally left out because it is Joplin-specific, ProseMirror-specific, or outside the product shape;
- what should become Galley backlog, including file drop/upload hooks, fenced-code completion, custom markdown extensions, KaTeX/Mermaid renderers, table controls, image resizing, and richer inline popovers.

### Update clean-room guidance

`CONTRIBUTING.md`, `README.md`, and the roadmap no longer describe a local `3rdparty/` source tree. The clean-room rule remains: behavior must be specified in prose and implemented from Galley tests and architecture, not translated from external code.

### Remove pre-rebrand leftovers

Neutrino image leftovers and stale worktree files are removed from the workspace. Historical release notes may still mention older package names where they describe actual history.

### Use Galley image assets in Storybook

Storybook markdown image demos use `assets/galley.png` for the mark and `assets/galley-color.png` for the project logo. The old generated `assets/img.png` and `src/stories/assets/` fixtures are removed.

### Release metadata

The package version is bumped to `0.7.0`, and `GALLEY_VERSION` is updated so the editor footer tooltip reports the new version.

## Acceptance Criteria

- `3rdparty/editor/` is not present in the repository.
- No Neutrino-named files remain in the working tree outside historical text.
- Storybook image examples render `assets/galley.png` and `assets/galley-color.png`.
- Old generated image fixtures are not present.
- Public docs frame this as a 0.7.0 cleanup milestone.
- `npm test`, `npm run lint`, `npm run build`, `npm run build:lib`, `npm run build-storybook`, and `git diff --check` pass.
