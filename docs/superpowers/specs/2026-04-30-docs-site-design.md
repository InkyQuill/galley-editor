# Galley Docs Site Design

## Goal

Create a public documentation website for Galley Editor using Astro Starlight. The site should be useful to React consumers who want to install Galley, understand its customization model, preview real examples, and wire advanced behavior such as renderers, toolbar slots, footer widgets, and future upload handlers.

## Architecture

The documentation app lives in `docs-site/` as a separate Astro project. This keeps Starlight's content collections, assets, and dependency graph separate from the library's Vite/React source tree while allowing root `docs:*` scripts to act as convenient entry points.

Existing markdown under root `docs/` remains engineering documentation: specs, plans, internal architecture, and audit notes. Public docs are copied or rewritten into Starlight pages under `docs-site/src/content/docs/`.

## Publishing

The default deployment target is GitLab Pages with namespace-in-path URLs:

```text
https://pages.inkyquill.net/inky/galley-editor
```

Astro receives this as `site = https://pages.inkyquill.net` and `base = /inky/galley-editor`. The docs app also supports a custom root domain later, for example:

```text
GALLEY_DOCS_SITE=https://galley.inkyquill.net
GALLEY_DOCS_BASE=
```

The GitLab CI `pages` job builds `docs-site/` and Storybook, then copies `docs-site/dist/` into the root `public/` artifact directory required by GitLab Pages. Storybook is copied into `public/storybook/`, so documentation and interactive examples are served from the same Pages instance.

## Information Architecture

Initial public pages:

- Overview
- Installation
- Quick Start
- Customization
- Plugins and Renderers
- Storybook
- API Reference
- Architecture
- Roadmap
- 0.7.0 Release Notes

The Starlight sidebar is explicit rather than autogenerated so release notes and reference pages can keep a stable order.

## Design

Use Starlight's default documentation ergonomics: readable content, built-in search, responsive sidebar, light/dark themes, and accessible navigation. Apply a small Galley theme layer with the transparent color logo, restrained ink/blue accents, and clean code/document styling. Keep the voice professional, direct, and lightly friendly. Avoid a custom marketing site until the API is closer to v1.0.

## Verification

The work is complete when:

- `npm --prefix docs-site run build` succeeds.
- `npm run build-storybook` succeeds.
- Root docs scripts work.
- The GitLab Pages job publishes Starlight at the Pages root and Storybook at `/storybook/`.
- The docs explain both default Pages deployment and custom-domain deployment.
