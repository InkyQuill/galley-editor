# v1.0 API Stabilization Design

## Goal

Stabilize Galley Editor's public API after the v0.8 and v0.9 feature milestones, expand integration examples, complete accessibility/package checks, and tag the first SemVer-stable release.

## Scope

v1.0 covers two roadmap entries:

- More Storybook examples for app integration patterns.
- Stabilized v1.0 API review.

It also carries forward the public-release work already described in `docs/specs/v1.0-public-release.md`: documentation polish, accessibility audit, package validation, release metadata, optional public mirror decision, and optional npmjs publish decision.

## API Review

Audit every public export from `src/components/index.ts`:

- component exports;
- hooks;
- command exports;
- plugin factories and rendering constants;
- public types;
- built-in plugins;
- keymap metadata.

For each export, decide one of:

- keep as stable v1 API;
- rename before v1;
- remove from public API;
- keep but mark experimental in docs.

Expected areas to review carefully:

- `ImageRenderer` shape after v0.8;
- table command names after v0.9;
- whether `HIDE_DECORATION` and `BLOCK_CURSOR_LINE_PROXIMITY` should be public;
- whether built-in plugin instances should remain exported individually;
- whether command argument shapes should use typed option objects instead of `unknown[]` through `execCommand`;
- whether `tabIndents` default should flip to `false`.

The result is documented in a new `docs/api-review.md` and reflected in `docs/api-reference.md`.

## Breaking Changes Before Freeze

v1.0 is the last planned breaking-change window before SemVer stability. Planned breaking changes:

- `tabIndents` defaults to `false` for keyboard accessibility.
- Any renamed/removed exports from the API review.
- Any refined v0.8/v0.9 APIs discovered during documentation and Storybook integration work.

Every breaking change must appear in `docs/migration-from-pre-1.0.md` and `CHANGELOG.md`.

## Storybook Integration Coverage

Expand Storybook from feature demos into integration patterns:

- controlled editor with save status;
- form integration with validation and submit;
- fake paste/drop upload workflow from v0.8;
- image metadata editor and resize renderer from v0.8;
- math and Mermaid renderer registration from v0.9;
- table editing toolbar and widget controls from v0.9;
- command palette pattern using `BUILTIN_COMMAND_NAMES`;
- custom toolbar/footer slots;
- read-only preview;
- accessible labeled editor;
- error boundary fallback;
- hook-first `useGalley` integration.

Stories should be practical app examples, not marketing layouts. They can share helper components to avoid bloating `GalleyEditor.stories.tsx`.

## Documentation

Update or create:

- `README.md` with stable v1 quick start and links.
- `docs/api-reference.md` from audited public exports.
- `docs/commands.md` with v0.8/v0.9 commands.
- `docs/plugins.md` with renderer examples and table plugin notes.
- `docs/styling.md` for current CSS variables and image/table metadata classes.
- `docs/accessibility.md` with audit checklist and results.
- `docs/migration-from-pre-1.0.md` for breaking changes.
- docs-site pages mirroring the stable public docs.

## Accessibility And Packaging

Complete the v1 release verification:

- `axe-core` reports no serious/critical issues for the demo or a representative Storybook page.
- default light/dark CSS variable pairs meet WCAG AA contrast.
- `aria-label` and `aria-labelledby` are supported on the editor wrapper.
- `tabIndents=false` is default and documented.
- `publint` and `@arethetypeswrong/cli` pass.
- packed package installs in a strict React + TypeScript consumer fixture.

Manual screen-reader checks are documented in `docs/accessibility.md` with tested OS/browser/screen-reader combinations.

## Release Decisions

Before tagging v1.0, the maintainer decides:

- whether to create a public GitHub mirror;
- whether to publish to npmjs in addition to GitLab Package Registry;
- what screenshot/GIF belongs in the README.

The implementation plan should include explicit stop points for these decisions.

## Out Of Scope

- New editor features beyond docs/examples/API cleanup.
- Marketing site redesign.
- SSR support.
- Bundled upload, math, or diagram services.
- Internationalization of built-in labels.

## Acceptance Criteria

- `docs/api-review.md` records a decision for every public export.
- `tabIndents` defaults to `false`, with migration docs and tests.
- Storybook covers the listed app integration patterns and builds in CI.
- Public docs and docs site reflect v1 API reality.
- Accessibility audit is documented and automated axe checks pass.
- Package validation with `publint`, `@arethetypeswrong/cli`, pack/install, lint, tests, build, library build, Storybook build, and docs build passes.
- `CHANGELOG.md` has a `1.0.0` section and pre-1.0 migration notes are complete.
