# v0.9 Advanced Authoring Design

## Goal

Improve Markdown authoring for advanced document features while keeping Galley dependency-light and markdown-first.

## Scope

v0.9 covers two roadmap entries:

- Markdown extension registration examples for KaTeX and Mermaid.
- Better table editing helpers.

The feature set should make advanced authoring practical for app integrators without bundling KaTeX, Mermaid, or a full table editor.

## Markdown Extension Examples

Galley already accepts CodeMirror extensions through `extensions` and Galley plugins through `plugins`. v0.9 formalizes this with examples and small helper exports where existing APIs are awkward.

Add examples, docs, and Storybook stories for:

- KaTeX-style math rendering from inline `$...$` and block `$$...$$` syntax.
- Mermaid rendering from fenced code blocks whose language is `mermaid`.

The package must not depend on `katex` or `mermaid`. Examples use consumer-supplied render functions:

```ts
type MathRenderer = (input: { source: string; display: boolean; theme: 'light' | 'dark' }) => HTMLElement;
type DiagramRenderer = (input: { source: string; language: string; theme: 'light' | 'dark' }) => HTMLElement | Promise<HTMLElement>;
```

If an async renderer is needed for Mermaid, the example owns the async state inside its widget/plugin. The core plugin factory remains synchronous unless a concrete core use case requires otherwise.

## Extension Packaging

Provide example files under Storybook/docs rather than exported production plugins:

- `MathRendererStory` demonstrates consumer-owned parsing and rendering.
- `MermaidRendererStory` demonstrates fenced-code replacement based on language.
- `docs/plugins.md` and the docs site explain the pattern and make clear that consumers install/render their preferred libraries.

If reusable utility code is extracted, it should be generic and dependency-free, such as a fenced-code language predicate or source-range helper.

## Table Editing Helpers

Add table commands that operate on the markdown table containing the selection:

- `normalizeTable()` pads rows and aligns separator cells with current alignments.
- `insertTableRowBefore()` and `insertTableRowAfter()`.
- `deleteTableRow()`.
- `insertTableColumnBefore()` and `insertTableColumnAfter()`.
- `deleteTableColumn()`.
- `setTableColumnAlignment(alignment: 'left' | 'center' | 'right' | null)`.

Commands return `false` when the selection is not inside a parseable GFM pipe table. Multi-cursor behavior applies each command once per distinct table.

The parser should live in a table-focused helper module shared by `src/plugins/tables.ts` and the command implementations. It should preserve simple cell text exactly where possible, while normalizing whitespace around pipe separators.

## Table Widget Controls

The existing inactive table widget can expose optional editing affordances when the editor is editable and in live mode:

- row add buttons near table top/bottom;
- column add buttons near column boundaries;
- alignment controls per header cell.

These controls call the table commands. They must be keyboard-accessible and have concise `aria-label` values. In `preview` or `editable={false}`, controls are hidden.

The controls are not a spreadsheet. Direct cell editing remains markdown-first: clicking into the table reveals source.

## Storybook

Add examples for:

- custom math renderer;
- Mermaid fenced-code renderer;
- command-driven table editing toolbar;
- visual table controls in live mode;
- disabled table controls in preview/read-only mode.

Examples should use lightweight placeholder renderers so Storybook builds without KaTeX or Mermaid dependencies.

## Out Of Scope

- Bundling KaTeX, Mermaid, Prism, Shiki, or any renderer runtime.
- Full Markdown parser extension support beyond documented CodeMirror/Galley plugin registration patterns.
- Rich cell editing inside rendered table widgets.
- Row/column drag reordering.
- Complex escaped-pipe table parsing beyond the existing GFM-style pipe table scope.

## Acceptance Criteria

- Docs and Storybook show how to register math and diagram renderers without adding runtime dependencies to Galley.
- Table commands work on simple GFM pipe tables and return `false` outside tables.
- Table command tests cover insert/delete row, insert/delete column, alignment changes, normalization, and multi-cursor duplicate-table protection.
- Table widget controls are accessible, editable-only, and call the same command path as external toolbar integrations.
- Existing table rendering behavior remains compatible with v0.7 except for the added controls when editable.
