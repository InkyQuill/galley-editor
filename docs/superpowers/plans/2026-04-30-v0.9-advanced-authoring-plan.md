# v0.9 Advanced Authoring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add dependency-free examples for math/diagram renderer registration and markdown-first table editing helpers.

**Architecture:** Keep optional renderers in Storybook/docs as consumer-owned integrations. Move table parsing into a shared helper and implement table commands against markdown source, then let the rendered table widget call those same commands for optional controls.

**Tech Stack:** TypeScript, CodeMirror 6, Lezer Markdown/GFM, React 19, Vitest/jsdom, Storybook 10.

---

## Task 2: Table Editing Commands

**Files:**
- Create: `src/commands/tableEditing.ts`
- Modify: `src/commands/index.ts`
- Modify: `src/types.ts`
- Test: `src/commands/tableEditing.test.ts`

- [ ] **Step 1: Write command tests**

Test these commands on a two-column table with selection in body cell `2`:

- `normalizeTable()` returns normalized markdown.
- `insertTableRowBefore()` inserts an empty row before current row.
- `insertTableRowAfter()` inserts an empty row after current row.
- `deleteTableRow()` deletes current body row.
- `insertTableColumnBefore()` inserts an empty column before current column.
- `insertTableColumnAfter()` inserts an empty column after current column.
- `deleteTableColumn()` deletes current column.
- `setTableColumnAlignment('center')` updates the separator.
- commands return `false` outside a table.

- [ ] **Step 2: Implement command helpers**

Create command functions with this shape:

```ts
import type { EditorView } from '@codemirror/view';
import { serializeMarkdownTable, tableAtSelection, tableColumnAtPosition, tableRowAtPosition, type TableAlignment } from '../table-markdown';

function replaceTable(view: EditorView, table: GalleyTable): boolean {
  const text = serializeMarkdownTable(table);
  view.dispatch({
    changes: { from: table.from, to: table.to, insert: text },
    selection: { anchor: Math.min(table.from + text.length, view.state.doc.length) },
    scrollIntoView: true,
  });
  return true;
}
```

Each exported command gets the current table, mutates a cloned table model, then calls `replaceTable`.

- [ ] **Step 3: Register commands**

Add command names to `BuiltinCommand`, `BUILTIN_COMMANDS`, component exports, and command docs metadata. Use exact names from the design: `normalizeTable`, `insertTableRowBefore`, `insertTableRowAfter`, `deleteTableRow`, `insertTableColumnBefore`, `insertTableColumnAfter`, `deleteTableColumn`, `setTableColumnAlignment`.
