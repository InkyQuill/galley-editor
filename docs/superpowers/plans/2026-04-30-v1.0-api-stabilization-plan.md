# v1.0 API Stabilization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Freeze the public API, expand integration examples, complete accessibility/docs/package validation, and prepare the first stable release.

**Architecture:** Treat v1.0 as a stabilization pass after v0.8 and v0.9. Audit exports first, apply final breaking changes, then update Storybook/docs and run release-grade verification before tagging.

**Tech Stack:** React 19, TypeScript, CodeMirror 6, Storybook 10, Astro Starlight docs, Vitest/jsdom, publint, @arethetypeswrong/cli.

---

## File Structure

- Create `docs/api-review.md` for explicit public export decisions.
- Modify `src/components/index.ts`, `src/types.ts`, `src/commands/index.ts`, and affected implementation files based on API decisions.
- Modify `src/components/GalleyEditor.test.tsx` and controller/keymap tests for default accessibility behavior.
- Modify or split `src/components/GalleyEditor.stories.tsx` into integration-focused story helpers.
- Create `docs/accessibility.md` and `docs/migration-from-pre-1.0.md`.
- Rewrite `README.md`, `docs/api-reference.md`, `docs/commands.md`, `docs/plugins.md`, `docs/styling.md`, and docs-site equivalents.
- Modify `.gitlab-ci.yml`, `package.json`, `src/version.ts`, `CHANGELOG.md`, and roadmap status.

## Task 2: Accessibility Defaults And ARIA

**Files:**
- Modify: `src/types.ts`
- Modify: `src/components/GalleyEditor.tsx`
- Modify: `src/controller.ts`
- Test: `src/components/GalleyEditor.test.tsx`, `src/controller.test.ts`

- [ ] **Step 1: Write failing accessibility tests**

Add tests that:

- render `<GalleyEditor aria-label="Markdown body" />` and assert the CodeMirror editor wrapper has that accessible name;
- render `<GalleyEditor aria-labelledby="editor-label" />` and assert the attribute is forwarded;
- construct default settings without `tabIndents` and assert Tab can move focus rather than indenting;
- render `tabIndents={true}` and assert list indentation still works.

- [ ] **Step 2: Add ARIA props**

Add to `GalleyEditorProps`:

```ts
'aria-label'?: string;
'aria-labelledby'?: string;
```

Pass these to the element that owns the editor shell and make sure CodeMirror remains focusable.

- [ ] **Step 3: Flip `tabIndents` default**

Change the default from `true` to `false` wherever settings are resolved. Keep `tabIndents={true}` behavior unchanged.

- [ ] **Step 4: Verify focused tests**

Run:

```bash
npm test -- src/components/GalleyEditor.test.tsx src/controller.test.ts
```

Expected: accessibility and existing wrapper/controller tests pass.
