Galley Editor v0.10.0 was released on June 27, 2026.

This release is mostly about making the full editor shell easier to place inside real application layouts. It adds a first-class fill layout, tightens the base stylesheet contract, and removes a few CSS behaviors that made whole-editor styling harder than it needed to be.

## Styling and Layout

### Fill fixed-height containers

`GalleyEditor` now accepts `layout="fill"` for fixed-height app panes.

```tsx
<section className="workspace-pane">
  <GalleyEditor
    value={value}
    onChange={setValue}
    layout="fill"
  />
</section>
```

```css
.workspace-pane {
  height: 100dvh;
}
```

In fill mode, Galley disables autosize behavior and uses flex layout so the CodeMirror editor fills the available space while the toolbar and footer stay in the shell. The base stylesheet now includes:

- `.ge-layout-fill`
- `.ge-editor-body`
- `.ge-layout-fill .cm-editor`
- `.ge-layout-fill .cm-scroller`
- `.ge-layout-fill .cm-content`

Use `layout="autosize"` when the editor should grow with its content. Use `layout="fill"` only when a parent element gives the editor a real height.

### Cleaner whole-editor focus styling

The default CSS no longer adds an extra persistent focus outline around `.cm-editor.cm-focused`. CodeMirror focus still works, but app teams can now style the shell, toolbar, and focus treatment without fighting a built-in editor outline.

### Single selection layer

The base stylesheet no longer adds a native `::selection` rule over CodeMirror's drawn selection layer. This avoids duplicate-looking selection backgrounds when the editor is focused and makes `--ge-color-selection` the single selection color hook.

### Comment token color variable

Comment-like syntax now has a dedicated theme variable:

```css
.my-editor {
  --ge-color-token-comment: #6b7280;
}

.my-editor[data-theme='dark'] {
  --ge-color-token-comment: #9ca3af;
}
```

Both rendered code comments and Lezer token classes such as `.tok-comment`, `.tok-meta`, and `.tok-url` use `--ge-color-token-comment`.

## Other Changes

- The autosize extension now clears inline scroller styles when it is removed or replaced.
- The package is marked client-compatible at the entrypoint for frameworks such as Next.js.
- The library postbuild script is portable across operating systems.
- Lockfile and dependency audit metadata were synchronized.
- Semantic release automation was added for future releases.
